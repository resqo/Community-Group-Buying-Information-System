# 社区团购管理信息系统功能与实现说明

## 一、系统完成情况概述

本系统已经按照“普通用户、团长、商家、系统管理员”四类角色完成了前后端分离实现。系统后端基于 Spring Boot + MyBatis + MySQL，前端基于 Vue 3 + Vue Router + Axios + Element Plus。

系统当前已经实现社区团购课程设计的主要业务闭环：

1. 用户注册、统一登录、按角色跳转。
2. 商品分类、商品浏览、商家商品维护、管理员商品审核。
3. 拼团活动创建、用户发起拼团、用户参与拼团、免拼购买。
4. 用户创建订单、模拟支付、生成支付流水。
5. 商家查看订单、发货到自提点。
6. 团长维护自提点、发送取件通知、取件码核销。
7. 用户申请退款、管理员审核退款、同步订单和支付状态。
8. 系统公告和用户通知查看。
9. 健康检查接口，用于确认后端和数据库是否在线。

## 四、数据库设计

系统使用 `isdemo` 数据库，共 10 张核心表：

| 表名 | 作用 |
|---|---|
| `user` | 用户、团长、商家、管理员账号 |
| `category` | 商品分类 |
| `product` | 商品信息、库存、价格、审核状态 |
| `group_activity` | 拼团活动规则 |
| `group_instance` | 用户发起的拼团实例 |
| `orders` | 订单、取件码、支付状态、自提状态 |
| `payment` | 支付流水 |
| `pickup_point` | 社区自提点 |
| `refund` | 退款售后 |
| `notice` | 系统公告、订单通知、取件通知、退款通知 |

建表和演示数据脚本位于：

```text
community_group_buy.session.sql
```

## 五、前后端连接方式

前端通过 Axios 封装统一请求对象，文件为：

```text
community-group-buy/frontend/community-group-buy-frontend/src/api/request.js
```

开发环境中，前端请求地址使用：

```text
/api
```

Vite 会把 `/api` 代理到后端：

```text
http://localhost:8080
```

代理配置位于：

```text
community-group-buy/frontend/community-group-buy-frontend/vite.config.js
```

示例：

```js
api.login(data)
```

实际请求流程：

```text
浏览器 -> http://localhost:5173/api/auth/login
Vite 代理 -> http://localhost:8080/api/auth/login
Spring Boot -> AuthController.login()
```

## 八、核心业务流程

### 8.1 普通购物流程

```text
用户登录
 -> 浏览商品
 -> 选择商品和自提点
 -> 创建订单
 -> 模拟支付
 -> 商家发货
 -> 团长发送取件通知
 -> 用户查看取件码
 -> 团长核销
 -> 订单完成
```

### 8.2 拼团购物流程

```text
管理员创建拼团活动
 -> 用户发起拼团
 -> 用户创建拼团订单
 -> 其他用户参与拼团
 -> 达到成团人数
 -> 用户支付
 -> 商家发货
 -> 团长核销
```

### 8.3 退款售后流程

```text
用户提交退款申请
 -> 管理员查看退款申请
 -> 管理员通过或拒绝
 -> 通过时订单状态变为已退款
 -> 支付状态变为已退款
 -> 系统发送退款通知
```

## 十、仍可继续扩展的功能

当前系统已经满足课程设计和基础业务演示要求。若后续继续完善，可以增加：

1. 后端 JWT 拦截器，严格校验接口角色权限。
2. 密码 BCrypt 加密存储，替代当前演示用明文密码。
3. 商品图片上传功能，替代手动填写图片地址。
4. 订单分页查询，适合数据量较大时使用。
5. 拼团失败自动退款定时任务。
6. 图表统计，例如日订单量、销售额趋势、热门商品排行。
7. 操作日志表，记录管理员审核、发货、退款等关键行为。
